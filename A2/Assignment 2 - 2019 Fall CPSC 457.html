Q3 skeleton

posted May 16, 2019, 8:44 PM by Pavol Federl   [ updated May 16, 2019, 8:44 PM ]

The code below scans the current directory for all files using

popen("find . -type f","r")

It then goes through each file and determines its length. It prints out each file together with its length, and finally reports the total size. You can modify this code to implement Q3 of this assignment.

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <sys/types.h>
#include <sys/stat.h>
#include <unistd.h>

#define MAX_FNAME_SIZE 512
#define MAX_FILES 1024

int main( int argc, char ** argv) {
  // handle command line arguments
  if( argc != 1) {
    fprintf(stderr, "I don't take command line arguments.\n");
    exit(-1);
  }
  // open 'find...'
  FILE * fp = popen( "find . -type f", "r");
  if( fp == NULL) {
    perror("popen failed:");
    exit(-1);
  }
  // read in all filenames
  char buff[MAX_FNAME_SIZE];
  int nFiles = 0;
  char * files[MAX_FILES];
  while(fgets(buff,MAX_FNAME_SIZE,fp)) {
    int len = strlen(buff) - 1;
    files[nFiles] = strndup(buff,len);
    nFiles ++;
  }
  fclose(fp);
  printf("Found %d files:\n", nFiles);
  // get file sizes for each file and sum them up
  long long totalSize = 0;
  struct stat st;
  for(int i = 0 ; i < nFiles ; i ++ ) {
    if( 0 != stat(files[i],&st)) {
      perror("stat failed:");
      exit(-1);
    }
    totalSize += st.st_size;
    printf("\t%s : %ld\n", files[i], st.st_size);
  }
  printf("Total size: %lld bytes.\n", totalSize);
  // clean up
  for(int i = 0; i < nFiles ; i ++ ) {
    free(files[i]);
  }
  // return success
  return 0;
}

Hint - overall solution for Q1

posted Oct 1, 2017, 11:32 PM by Pavol Federl   [ updated Feb 7, 2019, 11:15 AM ]

Your overall solution should look something like this:

find <params> | sort <params> | head <params> | awk <params>

You just need to find the right <params> for each of the commands...

Hint - sort

posted Oct 1, 2017, 11:30 PM by Pavol Federl   [ updated May 16, 2019, 8:36 PM ]

Examples from: http://www.theunixschool.com/2012/08/linux-sort-command-examples.html

Consider a file with multiple fields:

$ cat file
Linux,20
Unix,30
AIX,25
Linux,25
Solaris,10
HPUX,100

alphabetic sorting file on the basis of the 2nd field:

$ sort -t"," -k 2 file
Solaris,10
HPUX,100
Linux,20
AIX,25
Linux,25
Unix,30

numeric sorting file on the basis of the 2nd field: 


$ sort -t"," -k 2 -n file
Solaris,10
Linux,20
AIX,25
Linux,25
Unix,30
HPUX,100

numeric sorting file on the basis of the 2nd field, reversed order:

$ sort -t"," -k 2 -n -r file
HPUX,100
Unix,30
AIX,25
Linux,25
Linux,20
Solaris,10

Hint - awk

posted Oct 1, 2017, 11:28 PM by Pavol Federl   [ updated Oct 1, 2017, 11:28 PM ]

Print all lines in file.txt that are longer than 80 characters:
cat file.txt | awk 'length($0) > 80'
Print the sizes of each file in the current directory, and the total size:
ls -l files | awk '{ x += $5 ; print $5 } END { print "total bytes: " x }'

Hint - head

posted Oct 1, 2017, 11:27 PM by Pavol Federl   [ updated Oct 1, 2017, 11:27 PM ]

To print the first 3 lines of the file readme.txt:
$ head -3 readme.txt
Or you can use pipes:
$ cat readme.txt | head -3
Find all files in the current directory, but only print the first 10:
$ find . -type f | head -10

Hint - find

posted Oct 1, 2017, 11:26 PM by Pavol Federl   [ updated Oct 1, 2017, 11:26 PM ]

To find and print all files in the current directory:
$ find . -type f -print
To find all files that start with a prefix 'abc', and print their full paths
$ find . -type f -name 'abc*' -print
Find all files in the current directory and print them together with their sizes:
$ find . -type f -printf '%p %s\n'
